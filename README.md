# CloudHand-Mini-Term (云手原生控制终端)

> 极致安全的远程跨网段终端解决方案，彻底分离“控制面”与“执行面”，基于原生纯 Node.js 的系统托盘实现。

## 💡 架构理念：两端分离

CloudHand-Mini-Term 现已彻底重构为一个由**两部分组件**组成的非对称安全架构系统：

1. **公网桥接端 (Relay Server)**：放置在有公网 IP 的服务器上（或内网穿透入口）。它是一座没有任何底层终端执行能力的“空桥”，它只负责提供坚不可摧的暗黑密码墙以及 Web UI。
2. **本地守护托盘 (Node Tray App)**：部署在你要远程受控的物理机上。它没有复杂庞大的桌面 GUI（摒弃了厚重的 Tauri/Electron），而是作为极简的小图标常驻在 Windows 托盘。它每 5 分钟会变一次配对脸谱，并主动向桥接端发起连接撞击！

## 🛡️ 坚如磐石的安全堡垒

- **零知识哑巴屏墙**：公网访问时，绝不透露关于密码重置和交互的任何提示信息，暴力破解将毫无方向。
- **5 分钟动态沙漏口令**：本地托盘每五分钟生成一次仅限当前有效的配对码。只有当用户将此密码（通过安全信道，比如微信/OpenClaw聊天窗）私房传递并热更后，两者才会连通。
- **存活不死会话**：验证通过发行的 Token 和底层 WebSocket 长连免疫 24 小时过期机制。已经连上的合法用户执行再久也不会断流。
- **内网特权更新口**：重置密码的路由 `127.0.0.1` 被在网关级锁死，黑客永远找不到在公网上更改密码的 API。

## 🚀 部署与使用

### 第一步：开启公网中继器 
在你对外暴露的服务器进入 `openclaw-skill/` 目录运行桥接控制台：
```bash
cd openclaw-skill
npm install
# 初始可随便填一个临时防撞密码
node relay-server.js --pair-code=123456 --port=3456
```

### 第二步：启动本地托盘机
在你要被控制的本地电脑根目录下，直接静默运行主程序即可，它将隐没在右下角：
```bash
npm install
node server.js
```
> **托盘操作指南**：
> - 托盘可实时看见每 5 分钟换新的配对码（支持左键一键复制）。
> - 右击唤出【⚙️ 打开设置...】会用浏览器跳出一个超级轻便（跑在 127.0.0.1:9899 端口）的高级暗黑表单。
> - 在表单里填上你第一步中布的服务器地址，配置将被永久保存在 `settings.json` 当中，它会自动在后台打通连接。

### 第三步：热连通关
通过对话叫 OpenClaw (或者你在那台服务器自己发局域网请求) 执行：
```bash
curl -X POST http://127.0.0.1:3456/api/internal/paircode \
  -H "Content-Type: application/json" \
  -d '{"pairCode": "<你在右下角托盘刚刚复制的6位数>"}'
```
此时你电脑上一直疯狂撞门的托盘终端就会显示 `✅ 已连通接入 (就绪)`！
这个时候访问中继服务器提供给你的外部网页地址，在深邃的终端面板里开启黑客之旅吧！

## 📄 依赖 & 许可
- 服务端：`ws`, `express` (已移除), 本机原生 `crypto` 等。
- 托盘端：`systray2` (构建全平台无端界面), `clipboardy` (跨端剪切板通讯) 以及原生 `node-pty` 高性能底层。
