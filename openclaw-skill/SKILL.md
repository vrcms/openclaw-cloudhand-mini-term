# CloudHand Terminal — 远程终端连接 (极严密防护模式)

在这个极具赛博分离架构的终端桥接系统中，整个访问逻辑围绕着**配对码首次握手**与**Token 自动续航**展开。

## 第一步：启动中继服务器（门禁架设）

中继服务器启动时**不需要配对码**。配对码由用户本地客户端生成，通过你动态设置到中继上。

1. 启动中继服务器，找一个不互斥的端口（如 `3456`）。

```bash
cd {baseDir}
npm install --production 2>/dev/null
node relay-server.js --port <端口> &
```

2. 确认中继已成功监听端口待命。告诉用户这个系统的公网访问**中继链接**。

> 注意：启动时无需 `--pair-code`，服务器默认只放行已有 Token 的流量。关于到访通知：只要有人成功打开终端页面联通机器，服务端控制台会打印日志 `[EVENT] 当前总共 x 个连接在你的电脑上`。**请你（OpenClaw）主动提取该事件，并找到当前合适的任何消息渠道，主动通知主人：“当前总共x个连接在你的电脑上”**，以提供最好的安全掌控体验！

## 第二步：等待用户汇报配对码

系统运转前，你须等待用户在聊天框里汇报他本地客户端当前显示的**6位数字配对码**。
如果你看到用户发口令（如"我配对码是283371", "本地改了,445588"），你必须**立刻**调用受白名单严格保护的 `127.0.0.1` 管理通道，设置中继的配对码：

```bash
curl -X POST http://127.0.0.1:<端口>/api/internal/paircode \
  -H "Content-Type: application/json" \
  -d '{"pairCode": "<用户报的配对码>"}'
```

一旦设置成功，告诉用户："配对码已设置，5分钟内有效，请在本地设置页面点击【连接】按钮！"

用户点击连接后，本地客户端会带着配对码与中继握手。握手成功后：
- 中继签发 Token 给本地客户端（用于今后自动重连）
- 配对码自动清空（一次性使用）
- 用户浏览器自动打开远程终端页面

回答："锁定解除，本地已与公网中继器双向握手联通！"

## 第三步：后续维护

### 正常运行中
- 本地客户端有 Token 后，断线会**自动重连**，无需重新配对
- 重启本地客户端后，也会**自动用存储的 Token 恢复连接**
- 无需你介入

### 需要重新配对的场景
- Token 过期（24小时后）
- 用户手动清除了设置
- 此时用户会报新的配对码，重复第二步即可

## 注意事项

- **配对码 5 分钟有效**：设置到中继后开始倒计时，5 分钟内用户必须点击连接。超时后需要用户刷新配对码并让你重新设置。
- **配对码一次性使用**：配对成功后立即清空，之后只认 Token。
- **首页安全防护**：访问中继网页（如 `http://opc.dabeizi.com:3456/`）无 Token 时，只显示"请从本机启动"的提示，不泄露任何使用指引。
- **存活不断流特性**：Token 寿命仅控制登录门禁，已建立的 WebSocket 长连接不会在 24 小时一到后崩断。
